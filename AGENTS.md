# AGENTS.md

## Project

Go backend (Fiber v3) + HTML frontend for an RBAC-based auth/authorization management system ("KAS Bon"). Uses Casbin v3 for policy enforcement, GORM for ORM, Cobra for CLI, swag for Swagger docs.

## Commands

- `make dev` — regenerate swagger docs, then run with air (hot reload)
- `make go` — regenerate swagger docs, then `go run main.go serve`
- `make migrate` — run GORM AutoMigrate (idempotent)
- `make migrate-force` — drop all tables then re-migrate (destructive)
- `go run main.go policy --user <u> --role <r> --obj <o> --act <a>` — add a Casbin policy + grouping

Swagger docs must be regenerated before running (`swag init --parseDependency --parseInternal`). The `dev` and `go` make targets handle this automatically.

## Environment

Requires `.env` at project root (loaded via godotenv). Key vars:

```
PORT, DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME, DB_SCHEMA, JWT_SECRET
```

Database is **PostgreSQL only** (no SQLite/MySQL despite transitive deps).

## Architecture

### Entry points

- `main.go` → `cmd.Execute()` (Cobra root)
- `cmd/serve.go` — wires Fiber app, DB, Casbin, API routes, and web routes
- `cmd/migrate.go` — runs GORM AutoMigrate
- `cmd/policy.go` — CLI for adding Casbin policies

### Two routing layers

1. **API routes** (`/api/v1/*`) — `internal/v1/routes/routes.go` — REST JSON API with JWT auth + Casbin RBAC. Resources: users, roles, permissions, domains, apps, access-controls, token.
2. **Web routes** (root) — `web/routes/routes.go` — HTML pages (login, register, dashboard) with cookie-based auth middleware. Views are Go templates in `web/views/*.html`.

### Layering (internal/v1)

```
handlers → services → repositories → models
                ↘ usecases
```

- `models/` — GORM models and input DTOs
- `repositories/` — database queries
- `services/` — business logic, Casbin integration
- `handlers/` — Fiber HTTP handlers
- `middlewares/` — Casbin policy loader, JWT verification
- `usecases/` — additional business logic

### Shared code

- `internal/common/` — JWT generation, JSON response wrapper (`JSONResponse`), Casbin authorization helper
- `internal/configs/` — env setup, DB connection, Casbin setup
- `internal/enums/` — app constants, permission/role enums
- `pkg/arrays/` — shared array utilities

### RBAC model

Casbin policy definition: `sub, app, dom, obj, act` with role inheritance (`g = _, _, _`). Config at `rbac_model.conf`. Policies stored in DB via gorm-adapter. Superuser bypass: `all#fullAccess`.

### Generated code

- `docs/` — swagger.json, swagger.yaml, docs.go (generated by `swag init`). Do not edit manually.

## Conventions

- All API responses use `JSONResponse{items, isSuccess, message}`. Error responses pass `error` to `NewJSONResponse`.
- API auth: `Authorization: Bearer <JWT>` header. Multi-tenancy via `X-App-Id` and `X-Domain-Id` headers.
- Web auth: cookie-based (`web/middlewares/cookie_auth.middleware.go`).
- Passwords hashed with argon2id.
- No tests exist yet in this repo.
- Module path: `github.com/MarcelArt/kas-bon-v2` (Go 1.25).
- Swagger UI served at `/swagger`.
